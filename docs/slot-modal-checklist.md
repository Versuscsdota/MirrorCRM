# Чек-лист ручного тестирования: модальное окно слота

Дата: 2025-09-01
Компонент: `public/main.js` → `renderCalendar.openSlot()`

## Предусловия
- Под пользователем с ролью admin или root (для загрузки и удаления файлов).
- Есть хотя бы один слот в расписании.

## Сценарии

1. **Открытие слота**
   - Открыть слот кликом по ячейке в календаре.
   - Ожидаемо: модалка открывается, видны поля заметок, `Статус посещения (status2)`, блок «Вложения», кнопка «Регистрация» (для root/admin/interviewer), кнопки `status3`.

2. **Мультизагрузка файлов**
   - В поле «Вложения» выбрать сразу 2–3 файла разных типов: изображение, аудио, видео, PDF.
   - Нажать «Загрузить».
   - Ожидаемо: кнопка становится «Загрузка…», неактивна; после загрузки список обновляется, все файлы видны.

3. **Превью файлов**
   - Изображение: отображается миниатюра.
   - Аудио: отображается плеер `<audio controls>` и воспроизведение работает.
   - Видео: отображается плеер `<video controls>` и воспроизведение работает.
   - PDF/прочие: показывается иконка документа, без предпросмотра.

4. **Индивидуальное имя файла**
   - Очистить список выбранных файлов.
   - Выбрать один файл, ввести кастомное имя в поле «Название файла (для одиночной загрузки)» и загрузить.
   - Ожидаемо: в списке отображается заданное имя.

5. **Удаление файла**
   - Для роли root/admin нажать «Удалить» у любого файла.
   - Подтвердить (если есть подтверждение) и убедиться, что файл исчез из списка.

6. **Статус посещения (status2)**
   - Выбрать значения: «Пришла», «Не пришла», «Другое».
   - Для «Другое» ввести комментарий, сохранить изменения слота (если требуется кнопкой сохранения в модалке).
   - Ожидаемо: статус и комментарий сохраняются (проверить повторным открытием слота).

7. **Статус результата (status3)**
   - Нажимать на кнопки статуса (`thinking`, `reject_us`, `reject_candidate`).
   - Ожидаемо: статус обновляется и фиксируется при повторном открытии.

8. **Регистрация модели**
   - Нажать «Регистрация». Заполнить минимальные поля, загрузить документы/фото.
   - Отправить форму.
   - Ожидаемо: создаётся модель, открывается профиль модели; у слота `status3` изменяется на `registration`.

9. **Обновление истории**
   - При изменении статусов/времени/регистрации проверить блок «История» — новые записи отображаются с корректным временем и типом действия.

## Негативные проверки
- Загрузка больше 10 файлов за раз — ожидаемо отклоняется (ограничение бэкенда).
- Загрузка неподдерживаемого типа — отклоняется с ошибкой.
- Попытка удаления файла не-админом — кнопки «Удалить» нет.

## Визуальные проверки
- Модалка не выходит за экран (макс. ширина расширена до 680px).
- Карточки файлов (`.file-card`) имеют ховер и корректные отступы.

## Откат/очистка
- При необходимости удалить тестовые файлы через кнопку «Удалить».
